# Case Study 3: Nexus Technologies Group - Data Gap Challenges

## Scenario Context

This case study presents **Nexus Technologies Group Ltd**, a UK-headquartered technology conglomerate facing multiple data challenges in preparing its first GIR. You will practice applying estimation techniques and understanding how to handle incomplete data.

## Learning Objectives

By completing this case study, you will be able to:
- Apply the Data Gap Identification Framework
- Use estimation methodologies for missing SBIE data
- Handle mid-year acquisition data challenges
- Apply Administrative Guidance for historical acquisitions
- Document estimated figures appropriately

## Data Challenges

| Challenge | Affected Entities | Issue |
|-----------|-------------------|-------|
| **Mid-Year Acquisition** | DataFlow Solutions Inc. (USA) | Only 6 months of data |
| **Legacy ERP Systems** | Germany, Poland, Brazil | Cannot extract SBIE data |
| **Historical Acquisition** | Nexus India Pvt Ltd | No records of 2019 carrying values |
| **Incomplete Tax Records** | Nexus Singapore Pte Ltd | UTP reserve breakdown unavailable |

## Key Learning Points

1. **Mid-Year Acquisitions**: Use actual period data (not annualised); pro-rate SBIE inputs
2. **Legacy Systems**: Apply estimation with documented benchmarks
3. **Historical Acquisitions**: Rely on AG provisions for pre-Dec 2021 deals
4. **UTP Breakdown**: Use reasoned allocation based on available information
